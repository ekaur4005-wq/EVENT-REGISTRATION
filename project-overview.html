YUGANTAR – Igniting Minds, Inspiring Change



1. Project Overview


The QR-Based Event Registration & Attendance Management System is designed to streamline and automate the complete registration and attendance workflow for Yugantar 2025, a two-day ICAI student mega-conference.

The system integrates multiple platforms and technologies:
  • Google Sheets – Central database
  • Google Apps Script – Backend automation
  • QuickChart QR API – QR generation
  • GmailApp – Automated email dispatch
  • HTML/CSS/JS – Web dashboard
  • GitHub Pages – Hosting
  • AI Tools – Code optimization & debugging

The solution eliminates manual errors, reduces workload, and ensures real-time, reliable tracking of every attendee across venue, bus, lunch, and kit distribution checkpoints.

2. Objectives

  1. Create a fully automated registration workflow
  2. Generate unique QR codes for every registered student
  3. Automate email confirmations with embedded QR images
  4. Create a committee dashboard for real-time statistics
  5. Enable QR scanning for:
    • 15 Nov Venue Entry
    • 15 Nov Bus Entry
    • 15 Nov Lunch
    • 16 Nov Venue Entry
    • 16 Nov Bus Entry
    • 16 Nov Lunch
    • Kit Distribution
  6. Implement on-spot registration & instant QR issuance
  7. Provide secure admin access via password-based login
  8. Ensure smooth deployment via GitHub + Apps Script WebApp

4. Detailed Workflow

1. Project Overview


Step 1 — Data Collection

All registration data is collected via Google Form and automatically stored inside FINAL REG sheet.

Step 2 — Sheet Initialization

The setupSheet() function creates headers, formatting, and sets required column widths:

  • Student Name
  • NRO
  • Mobile
  • Email
  • QR Code
  • Email Sent?
  • Day-wise scan status (Venue, Bus, Lunch)
  • Kits Distribution
  • Timestamp

Step 3 — QR Code Automation

The generateQRCodes() function:

  • Converts student data into a JSON payload
  • Generates a scannable QuickChart QR URL
  • Inserts QR image directly into the sheet
  • Ensures no duplication

Step 4 — Personalized Email System

The sendInvitationEmails() function:

  • Generates a high-quality 500px QR image
  • Attaches QR as inline email image
  • Sends a professionally designed HTML email
  • Updates the “Email Sent?” status

Step 5 — QR Scanning System

The scanning module (handleQRScan()):

  • Decodes QR JSON
  • Matches NRO in sheet
  • Updates respective entry column
  • Prevents double scanning
  • Returns formatted response for the dashboard UI

Step 6 — On-Spot Registration

registerOnSpot():

  • Validates NRO
  • Creates new entries instantly
  • Generates QR & inserts into sheet
  • Sends successful registration response

Step 7 — Statistics & Monitoring

getStats():

  • Computes real-time totals
  • Day-wise attendance
  • Bus entries
  • Lunch count
  • Kits distributed

Step 8 — Dashboard Integration

Stats + QR scanning endpoints power the frontend dashboard hosted on GitHub or Google Sites.

Extensive tests were conducted:

Challenge Solution

QR JSON breaking in scanning Converted to compact JSON
Image not appearing in sheet Replaced formula-based QR with CellImage()
Dashboard caching issues Added real-time API-based data pulling
Duplicate function definitions Reorganized & cleaned script structure
AI-generated code inconsistencies Manual debugging + restructuring
